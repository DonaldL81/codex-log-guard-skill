# 项目初始化与文档维护指南

本文是本项目的开发文档入口，用于新会话接手、文档职责划分和后续维护。本文只维护文档使用规则，不记录具体代码流程或一次性排查过程。

## 1. 文档清单

| 文件 | 作用 | 职责边界 |
|---|---|---|
| `README.md` | 面向用户的产品说明 | 只写用户如何理解、安装和使用工具 |
| `SKILL.md` | 面向 Codex 的 Skill 调度说明 | 维护自然语言意图、命令入口、执行要求和安全边界 |
| `docs/00-项目初始化与文档维护指南.md` | 文档入口和维护规则 | 说明新会话读什么、文档怎么分工 |
| `docs/01-项目上下文.md` | 项目目标、技术方案、关键决策 | 说明项目是什么、为什么这样做 |
| `docs/02-项目逻辑说明.md` | 运行链路、核心流程、数据流、模块职责 | 说明项目怎么工作 |
| `docs/03-项目开发规范.md` | 运行、测试、发布、安全、文档同步规则 | 说明项目怎么开发和验证 |
| `docs/04-项目经验积累.md` | 项目专属踩坑和排查经验 | 记录只适用于本项目的经验 |
| `docs/05-通用开发规范与经验.md` | 通用规范入口 | 指向跨项目通用规范，不重复维护 |

## 2. 新会话阅读顺序

接手本项目时建议按顺序阅读：

1. `README.md`：理解用户视角的用途和使用方式。
2. `SKILL.md`：理解 Codex 如何调度本工具。
3. `docs/00-项目初始化与文档维护指南.md`：确认文档边界。
4. `docs/01-项目上下文.md`：理解项目目标和关键决策。
5. `docs/02-项目逻辑说明.md`：理解当前任务相关流程。
6. `docs/03-项目开发规范.md`：确认运行、验证、发布和安全要求。
7. 按任务追加读取相关源文件。

开始开发前必须检查 Git 状态，识别已有未提交改动。

## 3. 按任务追加读取

| 任务类型 | 追加读取 |
|---|---|
| 用户使用方式、文案、截图变化 | `README.md`、`SKILL.md` |
| Skill 调度、CLI 命令变化 | `SKILL.md`、`tools/CodexLogGuardCli.ps1`、`docs/02-项目逻辑说明.md` |
| GUI 交互变化 | `CodexLogGuard.ps1`、`README.md`、`docs/02-项目逻辑说明.md` |
| 拦截器、计数器、清理逻辑变化 | `lib/CodexLogGuardCore.psm1`、`docs/02-项目逻辑说明.md`、`docs/03-项目开发规范.md` |
| 安全边界、删除范围、隐私逻辑变化 | `SKILL.md`、`docs/03-项目开发规范.md`、通用开发规范 |
| 发布内容、忽略规则变化 | `.gitignore`、`docs/03-项目开发规范.md` |
| 新踩坑或排查路径 | `docs/04-项目经验积累.md` |

## 4. 文档职责边界

- README 面向普通用户，不写内部架构和开发细节。
- SKILL 面向 Codex 调度，优先短规则、命令映射、执行要求和安全边界。
- 项目上下文记录仍会影响后续开发判断的目标、决策和状态。
- 项目逻辑说明记录代码流程、数据流和模块职责。
- 项目开发规范记录本项目必须遵守的运行、验证、发布和安全规则。
- 项目经验积累记录本项目踩坑，不写跨项目通用规则。

同一类内容只维护在一个最合适的位置，避免 README、SKILL 和 docs 互相复制。

## 5. 文档同步规则

| 变化 | 需要同步 |
|---|---|
| 用户可见使用方式变化 | `README.md` |
| Codex 自然语言调用变化 | `SKILL.md` |
| 命令入口或参数变化 | `SKILL.md`、`README.md`、`docs/02-项目逻辑说明.md` |
| 核心流程或模块职责变化 | `docs/02-项目逻辑说明.md` |
| 安全、删除、发布、验证规则变化 | `docs/03-项目开发规范.md` |
| 项目目标、关键决策或技术债变化 | `docs/01-项目上下文.md` |
| 项目专属踩坑 | `docs/04-项目经验积累.md` |

## 6. 发布前文档检查

发布或推送前确认：

1. README 与当前用户入口、截图和命令一致。
2. SKILL 与当前 CLI 命令和安全边界一致。
3. docs 中没有过期的 GUI 独立版路径或已删除脚本。
4. 不提交 `_archive/`、`monitor-logs/`、SQLite 日志、备份文件、密钥或本地缓存。
5. 中文文档用显式 UTF-8 读取检查无乱码。
