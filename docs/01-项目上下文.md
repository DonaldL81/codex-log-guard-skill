# 项目上下文

本文维护 Codex 写盘异常检测工具的项目目标、使用场景、技术方案、关键决策和当前状态。

## 1. 项目目标

本项目是一个 Windows 本地工具，同时支持普通用户双击 GUI 和 Codex Skill 调度，用来检查和临时缓解 Codex 本地 `logs_2.sqlite` 异常写盘问题。

主要目标：

1. 让用户快速判断 Codex 是否存在异常写盘。
2. 提供一键安装和卸载 SQLite 日志拦截器的入口。
3. 提供固定时长写盘监测，并保存监测记录。
4. 支持 Codex 通过 Skill 直接执行检查、清理、安装保护和打开 GUI。
5. 在清理日志时避免要求 Codex 自身退出后继续执行命令，使用延迟清理助手完成流程。
6. 明确删除和隐私边界，避免误删用户文件或读取敏感内容。

不解决的问题：

1. 不修复 Codex 客户端内部日志写入实现。
2. 不替代官方补丁。
3. 不做长期常驻服务、自动更新或安装包。
4. 不读取用户消息正文、Codex 回复正文、API Key 或 Token。
5. 不处理 `.codex` 下除日志数据库和备份目录外的其他文件。

## 2. 用户和使用场景

| 使用者 | 主要场景 | 关键需求 |
|---|---|---|
| Windows 上的 Codex 用户 | 怀疑 Codex 导致磁盘写入偏高或变卡 | 快速检查并临时止血 |
| 使用 Codex 的开发者 | 想让 Codex 直接检查自身写盘状态 | 通过自然语言触发监测、清理和保护 |
| 不熟悉命令行的用户 | 需要可视化状态和按钮操作 | 双击 GUI 查看状态、安装拦截器、清理日志 |
| 需要留证分析的用户 | 观察任务运行期间写盘强度 | 保存监测 CSV 记录 |

## 3. 技术栈

| 层级 | 当前方案 | 说明 |
|---|---|---|
| GUI | PowerShell + Windows Forms | 轻量、免打包、适合临时 Windows 工具 |
| Skill/CLI | PowerShell CLI | Codex 调度 `tools/CodexLogGuardCli.ps1` |
| 核心逻辑 | PowerShell 模块 `lib/CodexLogGuardCore.psm1` | GUI 和 CLI 共用状态、拦截器、清理、采样逻辑 |
| SQLite 操作 | Python 标准库 `sqlite3` | 避免依赖 `sqlite3.exe` |
| 写盘监测 | Windows CIM 性能计数器 | 读取 Codex 进程 `IOWriteBytesPersec` |
| 数据保存 | CSV 监测记录 | 默认写入项目目录 `monitor-logs/` |
| 验证 | PowerShell 自检入口 | CLI、GUI、延迟清理助手各有自检 |

## 4. 架构概览

项目由以下部分组成：

1. `codex写盘异常检测.vbs`：普通用户双击入口，隐藏 PowerShell 控制台。
2. `CodexLogGuard.ps1`：GUI 主程序，负责界面、按钮事件、监测展示和用户交互。
3. `tools/CodexLogGuardCli.ps1`：命令行入口，供 Codex Skill 和手动 PowerShell 调用。
4. `tools/CodexLogGuardDeferredClean.ps1`：延迟清理助手，独立窗口等待用户退出 Codex 后清理并重装拦截器。
5. `lib/CodexLogGuardCore.psm1`：共用核心模块。
6. `SKILL.md`：Codex Skill 调度规则和安全边界。
7. `README.md`：面向用户的产品说明。

外部读取或处理的位置：

- `C:\Users\<user>\.codex\logs_2.sqlite`
- `C:\Users\<user>\.codex\logs_2.sqlite-wal`
- `C:\Users\<user>\.codex\logs_2.sqlite-shm`
- `C:\Users\<user>\.codex\logs_backup\*`
- `C:\Users\<user>\.codex\state_5.sqlite`
- `C:\Users\<user>\.codex\session_index.jsonl`
- 项目目录下的 `monitor-logs\*`

## 5. 运行和部署形态

| 形态 | 说明 | 入口 |
|---|---|---|
| GUI 本地运行 | 普通用户默认方式 | `codex写盘异常检测.vbs` |
| PowerShell 调试 GUI | 开发调试 | `powershell -NoProfile -ExecutionPolicy Bypass -STA -File .\CodexLogGuard.ps1` |
| Codex Skill | Codex 自然语言调度 | `SKILL.md` + `tools/CodexLogGuardCli.ps1` |
| CLI 手动调用 | 调试或自动化 | `powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\CodexLogGuardCli.ps1 <command>` |
| 容器运行 | 不支持 | 不适用 |
| 生产服务 | 不支持 | 不适用 |

## 6. 关键设计决策

| 决策 | 原因 | 影响 |
|---|---|---|
| 做成 GUI + Skill 一体仓库 | 同时服务普通用户和 Codex 调度 | README 面向用户，SKILL 面向 Codex |
| 核心逻辑放入 `lib` | 避免 GUI 和 CLI 两套实现漂移 | 后续状态、清理、拦截器逻辑优先改核心模块 |
| Skill 默认诊断跑 2 分钟监测 | 静态状态无法判断任务运行时写盘 | Codex 对话会等待监测结束再总结 |
| Skill 清理只走 `deferred-clean` | Codex 运行中无法“退出后继续执行当前 Skill” | 清理由独立助手等待退出、清理、重装拦截器 |
| 禁用 CLI `clean` | 避免误调用立即清理流程 | 清理入口统一为延迟助手和 GUI |
| 拦截器使用 SQLite 触发器 | 直接拦截 `logs` 表插入 | 不修复 Codex，只做临时止血 |
| 监测期间才启用计数拦截器 | 计数器自身会产生少量写入 | 平时保持纯拦截器减少写盘 |
| `_archive/` 不发布 | 存放本地素材和旧版备份 | `.gitignore` 忽略该目录 |

## 8. 已知技术债

| 问题 | 影响 | 后续处理建议 |
|---|---|---|
| GUI 和 CLI 写盘评估逻辑仍分别实现 | 后续阈值调整可能不一致 | 抽到 `lib` 中共用评估函数 |
| Windows 性能计数器可能卡住 | GUI 或 CLI 采样可能短时间等待 | 后续可加超时或降级采样 |
| PowerShell GUI 单文件仍较大 | 维护 UI 细节时定位成本高 | 保持小改，必要时再拆 UI 辅助函数 |
